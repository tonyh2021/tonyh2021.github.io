## copy and mutableCopy

Conclusions first, then code.

- copy: For a mutable object, calling `copy` returns a new immutable object, while the retain count of the original mutable object remains unchanged. For an immutable object, calling `copy` returns the original immutable object, and the retain count of that immutable object is incremented by one (equivalent to `retain`).

- mutableCopy: Regardless of whether the object is mutable or immutable, calling `mutableCopy` always returns a new mutable object, and the retain count of the original object remains unchanged.

```objc
// copy
NSMutableArray *mArr = [NSMutableArray arrayWithObjects:@1, @2, @3, nil];
NSArray *arr = [NSArray arrayWithObjects:@4, @5, @6, nil];

NSLog(@"Mutable object: %p", mArr);
printf("mArr retain count = %ld\n", CFGetRetainCount((__bridge CFTypeRef)(mArr)));

NSLog(@"Immutable object: %p", arr);
printf("arr retain count = %ld\n", CFGetRetainCount((__bridge CFTypeRef)(arr)));

id a = [mArr copy];

NSLog(@"Mutable object after copy: %p", a);

printf("mArr retain count = %ld\n", CFGetRetainCount((__bridge CFTypeRef)(mArr)));

id b = [arr copy];
NSLog(@"Immutable object after copy: %p", b);

printf("arr retain count = %ld\n", CFGetRetainCount((__bridge CFTypeRef)(arr)));
```

Extended notes:

Assigning between mutable and immutable objects can cause problems:

```objc
NSMutableArray *mArr = [NSMutableArray arrayWithObjects:@1, @2, @3, nil];
NSArray *arr = [NSArray arrayWithObjects:@4, @5, @6, nil];

NSLog(@"mArr address: %p, type: %@", mArr, NSStringFromClass([mArr class]));

NSArray *a = mArr;
// The following log shows that the declared type of a and its runtime type are different.
NSLog(@"a address: %p, type: %@", a, NSStringFromClass([a class]));

// Potential problem: you might assume a is immutable, but mArr can still modify it.
NSLog(@"a value: %@", a);
[mArr addObject:@4];
NSLog(@"a value after operating on mArr: %@", a);

NSMutableArray *b = mArr;
NSLog(@"b address: %p, type: %@", b, NSStringFromClass([b class]));

NSArray *c = arr;
NSLog(@"c address: %p, type: %@", c, NSStringFromClass([c class]));

NSMutableArray *d = arr;
// The line above already generates a compiler warning.
// The following log shows that the declared type of d and its runtime type are different.
NSLog(@"d address: %p, type: %@", d, NSStringFromClass([d class]));

// The following code crashes: -[__NSArrayI removeAllObjects]: unrecognized selector sent to instance 0x600000640e10
[d removeAllObjects];
```

## copy and strong in property declarations

First, one point must be clear: the `copy` modifier in a property declaration and the `copy` method call on an object are different concepts. The `copy` modifier in a property declaration means that the auto-generated setter will perform a copy operation on the incoming object.

```objc
// The following properties
@property (nonatomic, strong) NSArray *arrStrong;

@property (nonatomic, copy) NSArray *arrCopy;

@property (nonatomic, strong) NSMutableArray *mArrStrong;

@property (nonatomic, copy) NSMutableArray *mArrCopy;
```

```objc
NSMutableArray *mArr = [NSMutableArray arrayWithObjects:@1, @2, @3, nil];

NSLog(@"mArr address: %p, type: %@", mArr, NSStringFromClass([mArr class]));

self.arrStrong = mArr;

NSLog(@"mArr assigned to self.arrStrong, address: %p, type: %@", self.arrStrong, NSStringFromClass([self.arrStrong class]));

self.arrCopy = mArr;

NSLog(@"mArr assigned to self.arrCopy, address: %p, type: %@", self.arrCopy, NSStringFromClass([self.arrCopy class]));

self.mArrStrong = mArr;

NSLog(@"mArr assigned to self.mArrStrong, address: %p, type: %@", self.mArrStrong, NSStringFromClass([self.mArrStrong class]));

self.mArrCopy = mArr;

NSLog(@"mArr assigned to self.mArrCopy, address: %p, type: %@", self.mArrCopy, NSStringFromClass([self.mArrCopy class]));
```

The addresses in the output are actually unchanged! This is understandable: `arrStrong` and `mArrStrong` need no explanation. The `copy` in the setter does perform a copy, but as discussed in the previous section, copying an immutable object does not produce a new object.

Of course, if you use `_arrStrong = mArr` directly, the situation is the same as in the previous section.

Note: when you override the setter for a `copy`-modified property, you also need to perform a copy on the incoming parameter.

One more point, the following code:

```objc
NSString *str = @"123";
NSLog(@"Immutable object: %p, %@", str, NSStringFromClass([str class]));
NSString *str2 = @"123";
NSLog(@"Immutable object: %p, %@", str2, NSStringFromClass([str2 class]));
NSString *str3 = [@"123" copy];
NSLog(@"Immutable object: %p, %@", str3, NSStringFromClass([str3 class]));

NSString *str4 = [NSString stringWithFormat:@"123"];
NSLog(@"Immutable object: %p, %@", str4, NSStringFromClass([str4 class]));
NSString *str5 = [NSString stringWithFormat:@"%@", @"123"];
NSLog(@"Immutable object: %p, %@", str5, NSStringFromClass([str5 class]));
NSMutableString *mStr = [[NSMutableString alloc] initWithString:@"123"];
NSLog(@"Mutable object: %p, %@", mStr, NSStringFromClass([mStr class]));
NSString *str6 = [mStr copy];
NSLog(@"Immutable object: %p, %@", str6, NSStringFromClass([str6 class]));
```
